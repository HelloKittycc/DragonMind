## MVP Goal

DragonMind v0.1 validates a local-first personal digital advisor loop:

1. Capture a Spark quickly.
2. Preserve the original thought as append-only messages.
3. Interpret and connect related cognitive objects.
4. Surface the most important current issue through a Single Focus mobile UI.
5. Support lightweight judgment, stage progression, tasks, evidence, archive, and reminder cadence without introducing larger knowledge or multi-user systems.

## Implemented Features

- Spark capture.
- Data layer for `node`, `node_message`, `node_interpretation`, and `task`.
- Append-only `node_message` timeline.
- Automatic `spark_follow_up` task creation.
- Minimal `node_interpretation` creation.
- Relation graph with active relation deduplication.
- `related` relation normalization for logical undirected behavior.
- Stage progression:
  - Spark -> Reasoning
  - Reasoning -> Decision Prep
  - Decision Prep -> Decision
- `derived_from` relation creation for stage progression.
- Task creation and task status transitions.
- Reminder cadence using `Asia/Shanghai`.
- Task Revival.
- Evidence API for node and relation targets.
- Archive as lifecycle status, without moving or deleting historical data.
- Discovery Feed with related Spark and repeated Spark discovery.
- Runtime Importance query-time ordering; importance score is not stored.
- Single Focus mobile UI:
  - Today main issue
  - Judge flow
  - Possible pattern display model
  - Source modal for evidence/rationale items
  - Secondary Spark capture modal
- Workspace filters:
  - Inbox
  - Active
  - Decision
  - All
- Node Detail with cognitive timeline, task panel, relation panel, and evidence panel.

## Current Technical Stack

- Frontend: Next.js App Router
- Backend: FastAPI
- Database: SQLite
- Deployment posture: local-first, no cloud for v0.1
- Reminder timezone: `Asia/Shanghai`

## Current Database Objects

The v0.1 database contains:

- `node`
- `node_message`
- `node_interpretation`
- `task`
- `relation`
- `evidence`
- `schema_migrations`

No independent `decision`, `pattern`, `principle`, `possible_pattern`, `daily_brief`, `agent_recommendation`, or `judgment` tables are part of v0.1.

## Not Included In v0.1

DragonMind v0.1 does not include:

- Knowledge Ingestion
- RAG
- Vector DB
- Radius1
- Historical counterexample discovery
- Concept
- Insight
- Attention Pattern
- Outcome / Review
- Multi-user
- Cloud sync
- Conversation management
- Project management

## Known Limitations

- Local-first only; no production deployment posture.
- Single-user local database; no authentication or permission model.
- Discovery uses simple deterministic heuristics, not semantic retrieval.
- Runtime Importance is query-time and heuristic-based.
- Possible Pattern is frontend display only, not a persisted product object.
- Judge flow appends messages but does not create a separate decision or judgment table.
- Knowledge ingestion and external document ingestion are deferred.
- The UI is mobile-first; desktop polish is not the v0.1 target.

## Next Phase: v0.1.1 Knowledge Ingestion

The next planned phase is `v0.1.1 Knowledge Ingestion`.

Expected direction:

- Add a minimal ingestion path for user-provided knowledge materials.
- Keep the existing v0.1 cognitive object model stable.
- Avoid introducing Vector DB or RAG unless explicitly scoped and justified for v0.1.1.
- Preserve local-first behavior.
- Continue treating larger objects such as Concept, Insight, Attention Pattern, Outcome, and Review as out of scope unless a future spec promotes them into the architecture.
